# TDRホテル宿泊／レストラン予約 攻略情報

## 概要

TDRの希少価値が高く予約確保が困難なホテル宿泊とレストラン予約に役立つ情報を記載します。
具体的に言うと、宿泊ならミラコスタのテラスルーム・ハーバーグランドビュー／ハーバービュー、バルコニールーム・ハーバービュー、レストランならハイピリオン・ラウンジやドリーマーズ・ラウンジ辺りの予約確保を目標としている方に向けた情報です。

このサイトを見て何かを実践すれば必ずしも確実に予約が取れるということはありませんが、最低限度の知識や対策を把握することで争奪戦のスタートラインに立てる程度にはなれると考えております。
むしろ、このサイトの一番の存在意義は色々と情報に触れることで『自分も拾えるのではないか？』というモチベーションの維持、つまり**心が折れそうなときに自分を奮起させること**なのかも知れません。

## マシュマロにお返事

マシュマロを頂けたようで、ありがたいことですね。不定期に古い書き込みは消していきますが、悪しからずご了承ください。

* 確かに（恐らく予約完了からの）[キャンセル分の再販](https://www.tokyodisneyresort.jp/hotel/topics/info/operation/fsh_reservation.html)に関しては未定でしたね、[当該部分](./hotel/fsh_guide_0208.md#推測あるいは気休め)を書き直しておきます、ありがとうございます
    * 推測ではありますが、エラー落ちに関しては恐らくキャンセル扱いとは異なるので、当日のキャンセル拾い（本質的にはエラー枠拾い）争奪戦はやはり長丁場となると考えております、念のため…
    * とは言え、予約完走→キャンセルをすると当日復活が望み薄というのは、色々と悪用される可能性がないとも限らないような気はしますね
* 情報ありがとうございます、確かにそうですね、ボンボ予約の初手公開タイミングでボタンを押せるか勝負で待合室の時間は大きく変わってきそうですよね
    * ボンボの来店予約はエラーによる地獄モードというより、待合室から追い出す条件が予約サイトと異なる点が原因ではないかと推測します
    * 公式予約サイトならエラーでも成功でも10分経過後にTOPに戻ったら追い出されますので、何時間もエラーで滞在することは出来ないのですよね
* ページが消えたように見えたのでご心配おかけしてしまい恐縮です、ちょっとしたトラブルでした
    * 判る人向けに書くと、github pagesのActionsでworkflowが何故かin Progressで完全にハング（この時点でページが消えていた）してしまってキャンセルすら受け付けずデプロイが全く出来ない状態になっていました、仕方なくリポジトリを作り直したりもしたのですが今度はActions自体の有効化が出来ず完全にデプロイ手段がなくなっていました、今朝になってから試したら有効化が出来たようで復旧しました、時間が解決してくれたようですかね

[中の人にマシュマロで質問やメッセージを送る](https://marshmallow-qa.com/ecu5ejrsz5qmtuu)
