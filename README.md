# TDRホテル宿泊／レストラン予約 攻略情報

## 概要

TDRの希少価値が高く予約確保が困難なホテル宿泊とレストラン予約に役立つ情報を記載します。
具体的に言うと、宿泊ならミラコスタのテラスルーム・ハーバーグランドビュー／ハーバービュー、バルコニールーム・ハーバービュー、レストランならハイピリオン・ラウンジやドリーマーズ・ラウンジ辺りの予約確保を目標としている方に向けた情報です。

このサイトを見て何かを実践すれば必ずしも確実に予約が取れるということはありませんが、最低限度の知識や対策を把握することで争奪戦のスタートラインに立てる程度にはなれると考えております。
むしろ、このサイトの一番の存在意義は色々と情報に触れることで『自分も拾えるのではないか？』というモチベーションの維持、つまり**心が折れそうなときに自分を奮起させること**なのかも知れません。

## マシュマロにお返事

マシュマロを頂けたようで、ありがたいことですね。不定期に古い書き込みは消していきますが、悪しからずご了承ください。

* 過剰反応という表現が適切かはさておき、チートというのも線引きが難しいのですよね
    * 十把一絡げに**ブラウザ／アプリの手動操作以外は全て悪**と、正反対の**無差別級の何でもあり**は判りやすいですが、その間には広大なグレーゾーンが横たわっているのですよね
    * 例えば、URLをいじるのは良い≒好きなURLを叩いて良い≒通常ではあり得ない呼び出しをしても良い、とも言えますよね
    * URLをいじっても良いという人達でも、あり得ないURL呼び出しをして良いとまでは思わないでしょう
        * つまり、存在するかは不明ですがURLをいじって**叩けば確実に予約出来るURL**や**開放前に予約出来るURL**のような奇跡のURLを叩いても良い、という意見のつもりはないですよね
    * 逆に、通常操作のURLなら良いのか？ブラウザのブックマークは良いのか？という辺りのグレーゾーンもあるのですよね、例えば判りやすいのは予約を掴んでそのまま戻った後の画面のURL（自動で良い位置にスクロール）とかでしょうか
        * ブラウザの機能という意味で言うなら、Cookieクリアは良いのか、ブックマークレットは良いのか、デベロッパーツールは良いのか、という辺りも判断が難しいでしょうかね、さすがにデベロッパーツールはNGと言う人が多い予感ですが…（苦笑
    * 他にも、ただひたすら空き枠確認の内部処理を呼び出すスクリプトはNGなのか、機械的にブラウザを操作してその処理を呼び出すなら良いのか、手動で操作のみが正義としても毎秒複数回という頻度で叩いても手動なら良いのか等々、機械的な操作と手動操作に関しても操作自体の問題なのか、更新頻度（≒サーバー負荷）の問題なのかは判断が分かれそうでしょうかね
    * また、待合室の常時展開に関して言うなら、**1人で複数ブラウザを手動で起動して待合室を抜けた状態を維持**している人が多すぎるという問題の可能性もあるので、この操作（単にブラウザを複数起動しているだけ）をチートと言い切れる根拠というのも難しいですよね
    * ただ、ツールとやらを使わずに取ることが現実的に困難で、ほぼ全ての枠をツール利用者が確保しているようなケースでも、チートと呼ぶのは言い過ぎなのか、という視点もあるのですよね（ツール利用したから取れる訳では無いが、ツール利用しない人は取れない状態）
    * というような広大なグレーゾーンに対して中の人として個別に善悪を語るつもりはないのですが、どうしても『苦労して予約するからこそ価値がある』『楽してとった予約で楽しめるのか？』といった意見にだけは異を唱えたいですね
        * そんなことを言っているから日本の生産性は上がらず、景気も上向かず、給料も上がらず、何度もグランシャトーに泊まれないことになるのですよ
        * あくまでも**『効率化するのは当然、ただしルール／マナー違反はNG』という意見なら理解出来ます**が、勝手に苦労すること自体に価値を見出して、それを他者に押し付けて批判することで生産性を上げる努力を否定しないで欲しいところです
