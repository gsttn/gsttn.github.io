# TDRホテル宿泊／レストラン予約 攻略情報

## 概要

TDRの希少価値が高く予約確保が困難なホテル宿泊とレストラン予約に役立つ情報を記載します。
具体的に言うと、宿泊ならミラコスタのテラスルーム・ハーバーグランドビュー／ハーバービュー、バルコニールーム・ハーバービュー、レストランならハイピリオン・ラウンジやドリーマーズ・ラウンジ辺りの予約確保を目標としている方に向けた情報です。

このサイトを見て何かを実践すれば必ずしも確実に予約が取れるということはありませんが、最低限度の知識や対策を把握することで争奪戦のスタートラインに立てる程度にはなれると考えております。
むしろ、このサイトの一番の存在意義は色々と情報に触れることで『自分も拾えるのではないか？』というモチベーションの維持、つまり**心が折れそうなときに自分を奮起させること**なのかも知れません。

## マシュマロにお返事

マシュマロを頂けたようで、ありがたいことですね。不定期に古い書き込みは消していきますが、悪しからずご了承ください。

* 正直、待合室の細かい挙動は外からの観測だと限界があるので、中の人的に目的を達した時点であまり調べてはいないのですよね
    * ただ、ずっと待っていたのにある時点で突然待ち時間が伸びる、しかも複数の画面でほぼ同じようなタイミングで、というのはなかなか興味深いですね
    * 可能性が高いのは、公式側で設定をいじったパターンでしょうか、02/08は思ったより中は軽かったので途中で思い切り公式が絞った可能性はありそうかな、と思います
    * 次点で、ある時点から『本当に待っている人』が居る状態になったので、明確にサイト内に滞留している人が増え始めて待ち時間の残りに反映された等でしょうかね
    * いずれも細かい調査はしていないので推測の域は出ませんが…
* 黒エラーというのがどれなのか次第ではありますが、普通にページを開いて普通に予約しているなら、基本的にはアクセス集中くらいでしか発生しない気はしますね
    * ただ、長時間のキャンセル拾いをしているようなケースや、どこかで拾ってきたおかしなURL、ブラウザの戻る等を併用した想定外の画面遷移、というようなケースだと発生することもあるかも知れません
    * 不安ならCookieクリアして最初から普通の画面遷移で試してみるという辺りが無難かも知れませんが、大抵はそこまですることもないかと推測します
    * そもそも予約狙いの開放直後はアクセス集中しがちですし、そうでない時間帯はキャンセル拾いで長時間かけがちなので、原因は異なっても割と同じように見えるエラーが発生しやすい条件が揃っているのかも知れませんね
* ああ、出来るのですか、この戦術は…
    * ぶっちゃけ、このサイトを見ている人がそんなに多いとも思えませんし、ましてやTOPページを見ている人なんてさらに少ない（検索にヒットしにくい）ので心配には及ばない気もしますが、あとはこのネタを広める人が居るか次第でしょうかね
    * そもそも、普通に消してもgithub側に履歴が残るので完全には消せないのですよね、更新履歴も含めて全て消す方法がありそうな気もしますが…（直前までrebaseでまとめてforce pushとかだろうか…）
    * とは言え、『知っている人だけ一方的に得をする』『みんな知っているから中が高負荷な地獄モード』のどちらが良いかという判断は難しいところですよね
    * 極論するなら『IT系の知識がある人だけがシステムを組んで一方的に得をする』のはNGで、『公式サイトの穴を知っている人だけが特定の手順を利用して一方的に得をする』のは負荷対策等の観点で仕方ないからOKなのでしょうか？という話なのですよね
* 待合室の待機時間が途中で変わった（伸びた、しかも大幅に）という話は結構見かけますので、心中お察しします…
    * とは言え、環境の良し悪しによる差はあまりないので、基本的にはタイミングであり、突き詰めれば **運** でしょうかね
    * そして運をねじ伏せる方法は、このサイトのどこかに書いたような気がしますが、数の暴力ですね（この辺を[もう少し掘り下げて更新](./hotel/fsh_guide_0208.html#待合室への対策-20240209追記)しておきました）
    * さておき、そう言えばライトディナーの季節でしたっけ、相変わらず激しいキャンセル拾い争奪戦が繰り広げられているのでしょうかね…？（苦笑
